# FindKinWeb 前端和后台功能测试文档

## 一、 公共页面测试 (未登录用户)

### 1. 首页 (`index.jsp`)
- [ ] 首页可以正常加载，没有出现服务器错误。
- [ ] 页面整体布局美观，所有元素（导航栏、背景图、统计卡片、搜索表单、人员列表）均已应用新的 Bootstrap 样式。
- [ ] 未登录状态下，导航栏右上角显示 “登录” 和 “注册” 链接。
- [ ] 网站统计信息可以正常显示数字。
- [ ] 搜索表单的各个输入框和按钮样式正常。
- [ ] 下方的 “失踪人员列表” 以卡片形式展示，每张卡片包含照片、姓名、基本信息和发布时间。

### 2. 用户登录 (`login.jsp`)
- [ ] 从首页点击 “登录” 链接，可以跳转到新的登录页面。
- [ ] 登录页面布局美观，表单居中显示。
- [ ] 使用正确的管理员账户（如 `admin`/`admin`）可以成功登录。
- [ ] 使用正确的普通用户账户（如 `user1`/`password`）可以成功登录。
- [ ] 使用错误的用户名或密码登录时，页面会显示红色的错误提示信息。
- [ ] 登录页面下方有 “立即注册” 的链接，点击可跳转到注册页面。

### 3. 用户注册 (`register.jsp`)
- [ ] 从首页或登录页点击 “注册” 链接，可以跳转到新的注册页面。
- [ ] 注册页面布局美观，表单居中显示。
- [ ] 尝试注册一个全新的用户，所有字段填写完整，可以成功注册并自动跳转到登录页面。
- [ ] 尝试注册一个已存在的用户名，页面会显示错误提示信息。

## 二、 普通用户功能测试 (使用普通用户账户，如 `user1` 登录)

### 1. 登录后状态
- [ ] 登录成功后，页面跳转回首页。
- [ ] 首页导航栏右上角显示 “你好, [用户名]!”、“发布寻人启事” 和 “退出登录” 链接。
- [ ] 首页导航栏不应显示 “管理后台” 链接。
- [ ] 尝试直接通过 URL 访问管理后台页面 (如 `/admin/dashboard`)，应被自动重定向到登录页。

### 2. 发布寻人启事 (`publish.jsp`)
- [ ] 点击导航栏的 “发布寻人启事” 链接，可以跳转到发布页面。
- [ ] 发布页面布局美观，表单样式统一。
- [ ] 所有表单项（姓名、性别、年龄、失踪时间、地点、描述、照片、联系信息等）均可正常填写和选择。
- [ ] 成功填写所有必填项并提交后，可以成功发布，页面跳转到首页，并在列表中看到新发布的启事。
- [ ] 未登录状态下，尝试直接通过 URL 访问发布页面 (`/publish`)，应被自动重定向到登录页。

### 3. 退出登录
- [ ] 点击 “退出登录” 链接后，能成功退出。
- [ ] 退出后，首页导航栏恢复为 “登录” 和 “注册” 链接。

## 三、 管理员功能测试 (使用管理员账户，如 `admin` 登录)

### 1. 登录后状态
- [ ] 登录成功后，页面跳转回首页。
- [ ] 首页导航栏右上角除了普通用户的链接外，额外显示 “管理后台” 链接。

### 2. 管理后台 (`/admin/*`)
- [ ] 点击导航栏的 “管理后台” 链接，可以成功进入管理员主面板 (`dashboard.jsp`)。
- [ ] 管理主面板页面布局正常，左侧显示侧边栏导航，右侧显示内容。
- [ ] 侧边栏的 “主面板”、“用户管理”、“帖子管理” 链接可以正常点击和跳转。

### 3. 用户管理
- [ ] 从主面板或侧边栏点击 “用户管理”，可以进入用户列表页面 (`users.jsp`)。
- [ ] 用户列表以表格形式正确展示所有用户信息（ID, 用户名, 电话, 邮箱, 角色, 创建时间）。
- [ ] 点击任意用户后的 “编辑” 按钮，可以跳转到该用户的编辑页面 (`edit_user.jsp`)。
- [ ] 在用户编辑页面，可以修改用户的信息（如电话、角色），保存后生效。
- [ ] 在用户编辑页面，角色下拉菜单可以正确显示 “普通用户” 和 “管理员”。
- [ ] 在用户列表页面，点击任意用户后的 “删除” 按钮，在确认提示后，可以成功删除该用户。

### 4. 帖子管理
- [ ] 从主面板或侧边栏点击 “帖子管理”，可以进入寻人启事列表页面 (`posts.jsp`)。
- [ ] 帖子列表以表格形式正确展示所有启事信息（ID, 姓名, 性别, 状态等）。
- [ ] 帖子的 “状态” 字段能以不同颜色的徽章（如“未找到”为黄色，“已找到”为绿色）正确显示。
- [ ] 点击任意帖子后的 “编辑” 按钮，可以跳转到该帖子的编辑页面 (`edit_post.jsp`)。
- [ ] 在帖子编辑页面，可以修改帖子的信息（如姓名、地点、状态），保存后生效。
- [ ] 在帖子编辑页面，状态下拉菜单可以正确显示 “未找到” 和 “已找到”。
- [ ] 在帖子列表页面，点击任意帖子后的 “删除” 按钮，在确认提示后，可以成功删除该帖子。
